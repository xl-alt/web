API 使用指南

我们将复杂的调用过程模块化，为您提供清晰、易懂的使用说明。

1. API 接口地址

所有请求都应发送至以下 V1 接口地址。


// 推荐的主接口地址
const API_HOST = "https://happyapi.org/v1";

// 兼容备用地址 (一般无需使用)
// const API_HOST_COMPATIBLE = "https://happyapi.org/v1/chat/completions";

2. 请求示例 (cURL)

我们完全兼容 OpenAI 格式，这是一个标准的 cURL 请求示例。


# 向 chat/completions 端点发送 POST 请求
curl -X POST "$API_HOST/chat/completions" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {
        "role": "user",
        "content": "你好，请介绍一下你自己"
      }
    ],
    "stream": true
  }'

3. 绘画模型使用说明

① 对话式生成 (v1/chat/completions)

通过标准对话接口即可生成图片，支持图生图（叠图）。传图格式与 OpenAI 官方文档完全一致。

注意: 图片宽高比仅支持像素比。例如1024:1024，是指宽1024，高1024像素。

在提示词末尾（提示词使用英文效果更好哦！），使用英文中括号和冒号指定宽高比，例如：
帮我画一只宇航猫在月球漫步[1024:1024]

② 专业图片生成 (v1/images/generations)

使用此接口进行文生图。负载示例如下：



{
  "model": "flux.1-kontext-dev",           // 可选，指定模型的 ID，默认使用 "MODEL_ID_IMG2IMG"
  "prompt": "a cat", // 描述图片编辑的文本，例如：替换背景
  "size": "1024x1024",             // 可选，生成图片的尺寸，默认为 1024x1024
  "response_format": "url"       // 返回格式，支持 "url"（返回图片 URL）或 "b64_json"（返回 base64 字符串）
}

③ 专业图片编辑 (v1/images/edits)

支持对单张图片进行编辑。图片可以是 URL 或 Base64 字符串。负载示例如下：


{
  "model": "flux.1-kontext-dev",           // 可选，指定模型的 ID，默认使用 "MODEL_ID_IMG2IMG"
  "image": "https://xl-files.oss-cn-chengdu.aliyuncs.com/images/img-1756543093582-76007ced-e710-4bc3-92ea-f79ff9304750.jpg",  // 要编辑的图片，可以是 base64 字符串或 URL
  "prompt": "move to water", // 描述图片编辑的文本，例如：替换背景
  "size": "1024x1024"             // 可选，生成图片的尺寸，默认为 1024x102
  "response_format": "b64_json"   // 返回格式，支持 "url"（返回图片 URL）或 "b64_json"（返回 base64 字符串）
}

4. 语音 (TTS) 调用方法

通过 GET 请求调用，返回 MP3 文件。

请求地址:

https://fast-horse-44.deno.dev/v1/audio/speech

请求参数:

  • model: 声音模型 (如: zh-CN-YunxiNeural)
  • voice: 语速和语调 (格式: rate:0|pitch:0)
  • input: 要转换的文本

完整示例:

https://fast-horse-44.deno.dev/v1/audio/speech?model=zh-CN-YunxiNeural&voice=rate:0|pitch:0&input=你好，世界
